# В рамках дипломной работы разработанна дополнительная функциональность для проекта java-explore-with-me.

Фыбранный функционал "Comments" с некоторыми дополнениями.

Теперь у всех зарегестрированных пользователей появилась возможность добавлять комментарии к опубликовванным событиям. "Некоторым дополнением" является возможность зарегестрированного пользователя поставить "лайк" комментарию в виде оценки, далее рейтинг комментария вычисляется из AVG его оценок.

У администраторов приложения, есть функционал удаления комментариев, однако комментарий не удаляется совсем, а переходит в архивную таблицу "deleted_comments". 

Это дает возможность в случае спорных ситуаций, или иных нуждах (возможно какой-то аудит :-D) восстановить удаленный комментарий.

# End point'ы дополнительной функциональности:

                                                                PIVATE ACCESS
`POST events/{eventId}/users/{userId}/comments` - Оставить комментарий событию.

`PATCH events/{eventId}/users/{userId}/comments/{commentId}` - Обновить комментарий.

`DELETE events/{eventId}/users/{userId}/comments/{commentId}` - Удалить комментарий.

`GET events/users/{userId}/event/{eventId}/comments` - Получить все комментарии текущего пользователя(по конкретному событию) + from , size.

`POST users/{userId}/comments/{commentId}/like` -	Оставить лайк комментарию(оценку).

`PATCH users/{userId}/comments/{commentId}/like/{likeId}` - Обновить свой лайк(оценку).

`DELETE users/{userId}/comments/{commentId}/like/{likeId}` - Удалить свой лайк(оценку).


                                                                  PUBLIC ACCESS
                        
`GET /comments/{eventId}/all` - Посмотреть все комментарии у события. (с сортировкой “сначала новые” `?sort=NEW` или “сначала старые” `?sort=OLD`,  или по рейтингу `?sort=RATE`) + from, size. 

`GET /comments/{eventId}/search` - Поиск по комментариям события. (ключевые слова `?text="что ищем"` ) + from, size.

                                                                   ADMIN ACCESS

`DELETE admin/{adminId}/comments/{commentId}` - Удалить комментарий.(с указанием причины `?reason=Причина`)                                                                 

`GET admin/users/{userId}` - Получить все комментарии конкретного пользователя. + from, size.
                                                      
`GET admin/ deleted` - Получить удаленные комментарии. + from,size

`POST admin /return` - Восстановить комментарии. (Список айди комментариев для востановления , `важно что передать необходименно именно "старый" Id. ?ids=1,2`)

`GET admin/ deleted/{oldId}`  - Получить удаленный комментарий по "старому" id.

`GET /admin/{commentId}/like` - Посмотреть кто поставил лайки комментарию. + from, size

`GET admin/deleted/{oldId}` — Получить удалённый комментарий по "старому" id.


 
 ###                                                                Добавленные таблицы.

#### `comments`
- `id` (PK) — уникальный идентификатор
- `text` — текст комментария
- `event_id` (FK) — событие
- `author_id` (FK) — автор
- `created` — дата создания

#### `likes`
- `id` (PK) — уникальный идентификатор
- `rating` — оценка (1–5)
- `comment_id` (FK) — комментарий
- `liker_id` (FK) — пользователь

#### `deleted_comments`
- `id` (PK) — уникальный идентификатор
- `old_comment_id` — оригинальный ID комментария
- `text` — текст комментария
- `reason` — причина удаления
- `author_id` (FK) — автор
- `event_id` (FK) — событие
- `admin_id` (FK) — администратор
- `created` — дата создания комментария
- `deleted_at` — дата удаления
